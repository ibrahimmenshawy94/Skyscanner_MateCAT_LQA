# PT Style Guide for Skyscanner

## Critical Requirements

*   **Voice Priority**: **Helpful** is foundational. Playful and Geeky support this. If a conflict arises, choose **Helpful** first.
*   **Formality**: Use a friendly, respectful **3rd person tone**. **Never use "você"**. Avoid informal "tu" unless explicitly requested.
*   **Sustainability Claims**: **NEVER use** "sustainable", "eco", "green", "clean", "planet-friendly", or "guilt-free". Use comparatives (e.g., "emits 20% less carbon than...").
*   **"Best" & "Discount" Claims**:
    *   **Never** claim "Skyscanner offers" discounts. Use neutral phrasing: "You are offered a discount".
    *   **Do not** call deals "discounts" if referring to price drops/fluctuations.
    *   **Avoid "Best"** unless using a clear disclaimer explaining the metric. Use subjective alternatives like "epic prices" or "amazing deals".
*   **Gender**: **Use gender-neutral language** wherever possible. Avoid masculine defaults.

---

## 1. Tone of Voice

Our tone is a "Geeky Travel Pal":

*   **Playful**: Conversational, lively rhythm. Avoid corporate-speak, stiff phrasing, or "LOL-funny" attempts.
    *   *Tools*: Use rhyme, alliteration, metaphors, vivid adjectives sparingly. Clarity > Wordplay.
*   **Geeky**: Data-driven, humanized. Break complex info into simple steps. Sound like a knowledgeable friend, not a lecturer.
*   **Helpful**: Clear, simple, manageable.
    *   *Crisis/Bad News*: Drop playfulness. Apologize sincerely once. Focus on solutions. Do not be "cold".

## 2. Technical & Formatting Rules

### URLs
*   **Format**: `lowercase-only`.
*   **Spacing**: Use hyphens (-) only.
*   **Characters**: No special or extended characters (use basic Latin).

### Placeholders, Variables, & Tags
*   **Preservation**: **Do not translate, delete, or change** placeholders (e.g., `@@city@@`).
*   **Syntax**: Ensure sentences remain grammatically correct after populating variables. Rearrange if necessary for Portuguese syntax. Add spaces before/after tags if needed to prevent text collisions.

### Punctuation & Spacing
*   **Decimals**: Use a **comma** (e.g., `10,5`).
*   **Commas**: Do not use before "e" or "ou" unless the subject changes.
*   **Quotes**: Use **double straight quotes** (`""`). Use single quotes (`''`) for nested quotes.
*   **Ellipsis**: Capitalize following the ellipsis only if it starts a new sentence.
*   **Exclamation Marks**: Use sparingly. Avoid if they make the text seem forced or "ditzy".
*   **Full Stops**: Do not mirror English sentence length if it sounds unnatural. Split long sentences.
*   **Numbers**:
    *   1–9999: No spaces.
    *   10,000+: Use separators (e.g., `25 000`, `1 000 000`).
    *   Currencies: Symbol after number with space (e.g., `10 €`).

### Capitalization
*   **General**: Use **sentence case** for titles and CTAs.
*   **Titles**: Use the **infinitive form**.
*   **CTAs**: Use the **imperative form**.
*   **Colons**: Follow with a lowercase letter (unless a line break exists).
*   **Glossary/Product Names**: Use consistent sentence case for descriptive terms.

---

## 3. Linguistic Guidelines

### Inclusive Language
*   **Requirement**: Avoid stereotypes (gender, age, ability, background).
*   **Strategy**: Prefer neutral, precise terms over labels. Avoid the masculine-as-default.

### Anglicisms & Abbreviations
*   **Anglicisms**: Avoid unless widely expected (e.g., brand names, common digital terms).
*   **Abbreviations**: Use only if widely understood. Do not create new ones to save space.
*   **Plurals**: **Do not add an "s"** to abbreviations/acronyms in Portuguese.

### CTAs & Active Voice
*   **Voice**: Always use the **active voice**.
*   **Structure**: Keep CTAs short, direct, and imperative. Avoid passive constructions where the user is not the subject.

### Transliteration & Geo-Terms
*   **Consistency**: Use standard local exonyms if they exist. Use official sources for airport names.
*   **Airport Names**: Never add "airport" (it is added dynamically).
*   **Brand**: Use **"a Skyscanner"** (feminine form). Never translate "Sky" or "Scanner".

## 4. Content Type Specifications

| Content Type | Style Requirement |
| :--- | :--- |
| **UI** | Short, clear, action-focused. One idea per string. |
| **Marketing** | Fluid, inspirational, natural. Avoid literal source translation. |
| **SEO** | Natural language (no keyword stuffing). Descriptive headings. |
| **Google Ads** | Concise, clear benefits, direct CTA. No clickbait/unprovable superlatives. |
| **App Store** | Benefit-focused. No superlatives/guarantees. |
| **Editorial/Blog** | Descriptive, factual, practical. |
| **Legal** | Precise, neutral. **No jokes/emotions**. No extra promises. |

## 5. Do's and Don'ts (Summary)

*   **CTAs**:
    *   *Do*: "Reserve já"
    *   *Don't*: "Reservar Já" (Use imperative, not infinitive for CTA buttons)
*   **Structure**:
    *   *Do*: "Os 10 melhores destinos..."
    *   *Don't*: "Top 10 de destinos..."
*   **Formality**:
    *   *Do*: "Quando tivermos novidades, entraremos em contacto consigo"
    *   *Don't*: "Quando tivermos novidades, você será contactado" (Avoid "você").
*   **Clarity**:
    *   *Do*: "Compare voos para Tóquio e poupe tempo."
    *   *Don't*: "Veja aqui os melhores dos melhores voos..." (Avoid superlative repetition).
*   **Errors**:
    *   *Do*: "Ocorreu um erro. Tente novamente."
    *   *Don't*: "Ups… Não funcionou, mas não sabemos porquê." (Maintain helpful/professional tone).
*   **Lists/Formatting**:
    *   *Do*: "Escapadela urbana" (standard hyphenation/spacing rules).
    *   *Don't*: "escapadelaurbana" or "escapadela-urbana".
