# Norwegian Bokmål Style Guide for Skyscanner

## Critical Requirements
- **Tone Hierarchy**: Prioritize **Helpful** above all. If a choice must be made, be helpful first, then geeky, then playful.
- **Language**: Use informal **"du"** at all times. Never use the formal "De".
- **Legal Compliance**:
    - **Sustainability**: Never use absolute environmental claims (e.g., "green", "eco", "clean"). Use comparative data (e.g., "20% lower carbon").
    - **Discounts**: Never state "Skyscanner offers a discount". Use neutral phrasing like "Du får en rabatt" (from partners).
    - **"Best" Claims**: Avoid unless backed by a clear disclaimer on how "best" is measured.
- **Accuracy**: Placeholders (e.g., {city}, {date}) must **never** be translated, deleted, or changed.

---

## Tone of Voice: "The Geeky Travel Pal"

### Core Characteristics
- **Playful**: Conversational, light, rhythmic. Use rhyme, alliteration, and metaphors only if natural. **Clarity always > wordplay**. Avoid "fake-smart" corporate speak or trying to be "LOL-funny".
- **Geeky**: Enthusiastic about data and details. Explain patterns simply. Sound like a knowledgeable friend, not a lecturer.
- **Helpful**: Clear, honest, transparent.
    - **Bad News/Apologies**: Drop playfulness. Apologize once sincerely. Focus on solutions. Do not use cold phrases like "We apologize for the inconvenience."
    - **Structure**: "Pose question, then offer solution." Use short sentences.

---

## Technical & Formatting Guidelines

### URL Requirements
- **Format**: All lowercase, no special/extended characters (use basic Latin).
- **Separator**: Use standard hyphen `-` only. No spaces.

### Punctuation & Typography
- **Quotation Marks**: Always use **«»**. Never use “”.
- **Ellipsis**: Use one character (…), not three dots (...). Include space if sentence is cut off, no space if word is cut off.
- **Exclamation Marks**: Use sparingly. **Max one per sentence**. Avoid if it makes the text sound spammy.
- **Dashes**: Use en-dash (–), not em-dash (—).
- **Hyphens**: Use short hyphen (-) for compound words.
- **Full Stops**: Use at end of sentences. **Exclude** from buttons, menus, and titles.

### Capitalization
- **Sentence Case**: Use for titles and CTAs. Capitalize first word and proper nouns only.
- **Months/Days**: Always lowercase.
- **Colon**: Capitalize the following letter **only if** what follows is a complete sentence.
- **Product/Feature Names**: Use sentence case unless branded. Keep consistent (e.g., don't mix "Price Alerts" and "price alerts").

### Inclusivity
- **Neutrality**: Avoid stereotypes regarding gender, family, age, ability, or background.
- **Labels**: Focus on traveler needs, not labels. Use gender-neutral language.

---

## Language-Specific Rules

### Punctuation (Norwegian Standards)
- **Commas**: Use according to Språkrådet/Vinje. No Oxford comma. Comma after subordinate clauses. Comma before "men".
- **Active Voice**: Use imperative for CTAs. Avoid passive constructions.

### Anglicisms
- Avoid unless widely expected (e.g., "online" -> "på nettet", "sjekke" -> "se nærmere på", "matche" -> "tilsvarer"). Use Norwegian equivalents.

### Possessives
- English uses possessives more frequently; evaluate necessity. Place after the noun unless contrasting (e.g., "Dette er sykkelen din" vs "Dette er din sykkel").

### Airports & Geography
- **Airport Names**: Never add "flyplass" (airport) to the name; this is handled by placeholders.
- **Transliteration**: Use official/established local names. Do not invent spellings.

---

## Content-Specific Guidelines

### User Interface (UI)
- Prioritize extreme brevity and clarity over playfulness. One idea per string.

### Marketing
- Creative, natural, fluid. Focus on traveler benefits (saving time/money).

### SEO & Google Ads
- **SEO**: Natural language, no keyword stuffing.
- **Ads**: Concise, benefits-focused. Avoid superlatives ("best", "cheapest ever").

### Legal/Terms
- Neutral, precise. No emotional language. No interpretations not in source.

### Sustainability/Greenwashing (Mandatory)
- **Do not use**: "Sustainable", "eco", "green", "clean", "planet-friendly", "guilt-free".
- **Do use**: Comparative statements with data (e.g., "X emits 20% less carbon than Y").

## Translation Do’s & Don’ts Summary

| Feature | DO | DON'T |
| :--- | :--- | :--- |
| **Tone** | Helpful, conversational, clear. | Overly formal or "LOL-funny". |
| **CTA** | Imperative, short, active. | Passive, vague, long phrases. |
| **"Discount"** | "Du får en rabatt" (neutral). | "Vi tilbyr rabatt" (legal issue). |
| **Sustainability** | "20% lavere utslipp". | "Miljøvennlig flyreise". |
| **Airport Names** | "Paris Charles de Gaulle". | "Paris Charles de Gaulle flyplass". |
| **Quotes** | «Eksempel». | "Eksempel". |
| **Exclamation** | Gratulerer! | Gratulerer!!! |
| **Abbreviations** | Only if widely known. | Creating new ones to save space. |
| **Possessive** | "Sykkelen din". | Overusing "din/ditt" where unnecessary. |
| **Capitalization** | "Reis på en mandag." | "Reis på en Mandag." |

---

## Glossary & Terminology Logic
- **Always check**: Consistent capitalization across UI and copy.
- **Feature Names**: Must be clear and natural. If an English term is standard in the market, use it; otherwise, use the natural Norwegian equivalent (e.g., "pakkereiser" for package trips).
- **Variables**: **Never** translate or edit variables (e.g., `{price}`, `{city}`). Ensure spaces exist before/after tags to avoid concatenation.
