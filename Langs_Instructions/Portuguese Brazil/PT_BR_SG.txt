# Portuguese (Brazil) Style Guide for Skyscanner

## Critical Requirements

*   **Tone priority**: Helpful > Geeky > Playful. If a conflict arises, choose **Helpful**.
*   **Tone of voice**: Act like a knowledgeable, friendly travel pal. Use **"você" / "vocês"** (never "tu").
*   **Active voice**: Use for all headlines and CTAs.
*   **Clarity**: Prefer clarity over wordplay (rhyme, puns, alliteration). If it sounds unnatural, drop it.
*   **Accuracy**: Never make unsubstantiated claims (e.g., "cheapest," "best," "guaranteed").
*   **Format**: Use **sentence case** for titles and CTAs.
*   **Placeholders**: Never translate, delete, or modify tags (e.g., `@@city@@`).

## Brand Voice

### 1. Playful
- Conversational tone; treat the user like a friend.
- Create rhythm; avoid flat/robotic text.
- Use: Rhymes, alliteration, metaphors, vivid adjectives sparingly.
- **Avoid**: "Fake-smart" corporate speak, over-the-top "LOL" humor, and stiff phrases.
- **Constraints**: If a device (pun, rhyme) sounds forced in PT-BR, omit it. No "pinch-me-this-is-real" style long adjectives.

### 2. Geeky
- Enthusiastic but controlled; sound like a knowledgeable friend, not a lecturer.
- Break complex data into simple steps.
- Show expertise by explaining details humanely.

### 3. Helpful
- **Foundation**: Keep complex info simple, confusing situations clear, and overwhelming choices manageable.
- **Bad news**: Drop playfulness, apologize sincerely once, focus on solutions. Avoid cold phrases like "We apologize for the inconvenience."
- **Structure**: "Pose question, offer solution." Keep sentences short.

## Language-Specific Translation Guidelines

### Punctuation & Spacing
- **Commas**: Do not overuse. Split long sentences to avoid confusion.
- **Ellipsis**: Use only if it fits the playful tone; replace with a period/comma if it feels unnatural.
- **Exclamation marks**: Use sparingly. Okay for CTAs/dialogue; remove if they make text sound overexcited.
- **Quotes**: Always use straight double quotes (`"example"`) first. Use straight single quotes (`'example'`) only for nested terms.
- **Ampersands (&)**: Translate to "e" (e.g., "Voos e hotéis"), unless part of a brand name (e.g., "Dolce & Gabbana").
- **Colons**: Do not capitalize the word after a colon (unless a line break follows).
- **Semicolons**: Avoid. Use to clarify only if absolutely necessary; never for short lists.
- **Slashes**: No spaces before or after (e.g., `dispositivo móvel/celular`).
- **Currency**: Use a non-breaking space between symbol and value (e.g., `US$ 100`).

### Capitalization
- **Titles/CTAs**: Sentence case only.
- **Dates**:
    - Full: "11 de janeiro de 2026" (Day of week lowercase unless start of sentence).
    - 1st of month: "1º de fevereiro".
    - Abbreviated: "11/1/2026" (No leading zeros, use slashes).
- **Product/Feature names**: Consistent across UI and copy. Use sentence case.

### Addressing Users (T/V Policy)
- Use **"você" / "vocês"** at all times.
- Avoid "tu" unless required by strict legal text or specific marketing audiences.

### CTAs & Active Voice
- **Active voice** is mandatory for headlines and CTAs.
- Buttons:
    - Single word: Infinitive (e.g., "Reservar").
    - Multi-word: Imperative (e.g., "Reserve já").

## Technical & Content Specifications

### URLs
- No spaces (use hyphens `-`).
- All lowercase.
- No special/extended characters.

### Placeholders/Variables
- Keep exactly as in source. 
- Ensure grammatical correctness in PT-BR.
- If a preposition issue arises, use a workaround (e.g., `@@city@@: os preços de hotéis baixaram`) rather than risking ungrammatical phrases.

### Geo-terms & Airports
- Use official local names (verify via Google Maps/Wikipedia).
- **Do not duplicate**: If the name includes "Aeroporto", do not add it again (e.g., use "Aeroporto Charles de Gaulle", not "Aeroporto Aeroporto...").

### Sustainability
- **Avoid greenwashing**: No vague terms (green, eco, clean, planet-friendly).
- **Be factual**: Base claims on data.
- **Use comparatives**: Say "X emits 20% less carbon than Y", never "X is the greenest flight".

### Formatting & Style
- **Inclusive language**: Avoid gendered assumptions, clichés, or stereotypes about age/ability.
- **Anglicisms**: Use only if widely understood and expected (e.g., brand names).
- **Abbreviations**: Explain on first use if not standard (e.g., airport codes).
- **Emphasis**: Use bold/italics. **Never** use ALL CAPS.

## Content Type Rules

| Type | Tone/Strategy |
| :--- | :--- |
| **UI** | Short, clear, action-focused. One idea per string. |
| **Marketing** | Fluid, inspirational, natural. Benefit-focused. |
| **SEO** | Natural language, descriptive headings. No keyword stuffing. |
| **Google Ads** | Concise, benefits-driven, direct CTA. No superlatives/clickbait. |
| **App Store** | Simple, highlight benefits, no superlatives/guarantees. |
| **Editorial/Blog** | Warmer, descriptive, factual, practical. |
| **Emails/Push** | Short subject lines, clear next steps. |
| **Legal/Policies** | Precise, neutral, no emotion/jokes. Accuracy first. |

## Important Prohibitions
- **"Best" claims**: Do not use "best price" or "best deal" without a clear disclaimer or objective sorting mechanism. Use "epic prices" or "amazing deals" if subjective.
- **Discounts**: Never claim "Skyscanner offers a discount". It is always a partner offering it. Use neutral phrasing like "You are offered a discount."
- **Stereotypes**: Avoid any jokes or idioms relying on nationality, gender, age, or background.

## Glossary & Terminology Checklist
- Use local equivalents for "deal", "trip", "offer" if natural.
- **Do not** mix English/PT-BR in phrases (e.g., no "flight deals super baratos").
- **Consistency**: Keep UI labels consistent (e.g., always "Alerta de Preço", never "Monitoramento de Preço").

## Representative Examples (Do vs. Don't)

**Formatting**
- *Do*: "Garanta o carro que você mais deseja"
- *Don't*: "Garanta o carro que você MAIS deseja" (Avoid ALL CAPS).

**Buttons**
- *Do*: "Reservar" (Infinitive for 1 word)
- *Don't*: "Reserve"

**Placeholders**
- *Do*: "@@city@@: os preços de hotéis baixaram"
- *Don't*: "Os preços de hotéis em @@city@@ baixaram" (Risk of grammatical error).

**Sustainability**
- *Do*: "Este voo emite 20% menos carbono..."
- *Don't*: "Este é o voo mais sustentável."

**Clarity**
- *Do*: "Precisa de ajuda? Entre em contato..."
- *Don't*: "Entre em contato... caso precise de ajude" (Poor grammar/typo).
