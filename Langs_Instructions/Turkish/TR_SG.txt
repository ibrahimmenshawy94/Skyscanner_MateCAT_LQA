# TR Style Guide for Skyscanner

## Critical Requirements

**Tone of Voice (Geeky Travel Pal)**:
- **Prioritize Helpful > Geeky > Playful**.
- **Tone**: Playful/energetic but human. Use "pose question, offer solution" structure.
- **Formality**: Use the **Formal "Siz"** (second person plural) for all consumer-facing content (transactional/data sensitivity).
- **Mandatory Imperative**: Use suffix `-ın` or `-in`. **Strictly avoid** archaic/bureaucratic forms (`-ınız`, `-iniz`) and informal (`sen`) address.
- **Inclusive Language**: Avoid stereotypes (gender, age, family, ability). Use neutral/precise terms. Avoid visual-only verbs ("bakın", "görün"); use "gezinin", "açın", "gidin".

## Technical Specifications

### URLs
- **Format**: All lowercase, no spaces (use hyphens), no extended/special characters (replace with base Latin).
- **Non-Latin Scripts**: Keep English URLs for non-Latin languages.

### Placeholders/Tags
- **Never translate or change**: Keep exact syntax (e.g., `@@city@@`).
- **Spacing**: Ensure spaces around tags to prevent text gluing.
- **Grammar Workaround**: Rewrite the sentence structure if the tag placement forces incorrect grammar.

## Linguistic Guidelines

### Punctuation
- **Commas**: Minimize. **Do not use** after conditional clauses (`-se`/`-sa`) or adverbial clauses (`-iken`, `-dığında`, `-dıkça`).
- **Exclamation Marks**: Use sparingly. Do not use for general enthusiasm. Avoid `?!` or `!?`.
- **Ellipsis**: Exactly 3 dots (`…`). No space before.
- **Quotation Marks**: Use double straight (`" "`) for standard/UI/titles. Use single straight (`' '`) for nested quotes. Inflectional suffixes attach directly to the closing quote (no space).
- **Hyphens**: Avoid (except `e-posta`, `e-ticaret`). Ranges: Use En dash (`–`) without spaces.
- **Em Dash**: Avoid. Split into sentences or use commas/colons.
- **Full Stops**: Single space after. Do not use in headings/titles. Place inside parentheses if the sentence is complete.

### Capitalization
- **Style**: Sentence case (first word + proper nouns capitalized). Avoid Title Case.
- **Colons**: Capitalize if followed by a full sentence/proper noun; lowercase if followed by a fragment/list.
- **List Items**: Capitalize first word. End with period if full sentence.

### Date/Time Formatting
- **Short Format**: `DD.MM.YYYY` (e.g., 07.01.2018).
- **Long Format**: Day Number + Month Name + Year (e.g., 12 Ocak 2016).

## Content Type Guidelines

- **UI**: Short, clear, action-focused. One idea per string.
- **Marketing**: Natural, fluid, benefits-focused. Avoid "translation-ese".
- **SEO**: Natural language, no keyword stuffing.
- **Ads/App Store**: Concise, clear benefits, direct CTAs. No clickbait/unsubstantiated superlatives.
- **Legal**: Neutral, precise, no emotional language or jokes.

## Brand & Terminology Rules

- **Skyscanner**: Keep brand name as is. Do not translate.
- **"Best"**: Never use "Best" unless a disclaimer explains how it is measured. Use subjective alternatives (e.g., "epic prices") if prohibited.
- **"Discount"**: **Never** imply "We offer a discount". Use neutral phrasing ("You are offered a discount"). Never refer to price drops/deals as discounts.
- **Sustainability**: Avoid greenwashing. Use comparatives ("daha çevre dostu") rather than absolutes ("en ekolojik"). No unverified claims ("doğal", "temiz"). Factual data only.
- **Transliteration**: Use established local exonyms (e.g., "Türkiye"). Use official transliteration for non-Latin script airports. Do not manually add "Havalimanı" if the placeholder already includes it.

## Linguistic Usage & Examples

### Correct Usage Table
| Feature | Rule |
| :--- | :--- |
| **Commas** | Avoid after conditional clauses (e.g., "...isterseniz oturum açarak...") |
| **Em Dash** | Split into two sentences or use punctuation. |
| **"Best"** | Use only with mandatory disclaimer. |
| **Sustainability** | Use "düşük emisyonlu" instead of "ekolojik". |
| **UI Verbs** | Use "gezinin"/"açın" instead of "bakın". |
| **Capitalization** | Use sentence case for feature/product names. |

### Forbidden Styles
- Do not use corporate/stiff language.
- Do not use archaic formal imperatives (e.g., "yapınız").
- Do not use "fake-funny" marketing copy.
- Do not add "Havalimanı" if it causes duplication with the placeholder.
- Do not use "discount" or "indirim" for price fluctuations or "deals".

### Cultural/Imagery Sensitivity
- **Flag for review**: Culturally inappropriate imagery (clothing, food, gestures), religious references (pilgrimage sites), or sensitive political/ethnic content.
- **Symbols**: Avoid `&`, `#`, `@` in running text; use equivalents (e.g., "ve").

### Translation Logic
- **Playfulness**: Use only if natural (rhymes, alliteration). Abandon if forced.
- **Clarity**: Always prioritize accuracy and clarity over style/wordplay.
- **Reference**: Follow TDK standards (tdk.gov.tr).
