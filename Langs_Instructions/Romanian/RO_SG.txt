# Romanian Style Guide for Skyscanner

## Critical Requirements
**Tone Hierarchy**: **Helpful > Geeky > Playful**. If a choice must be made, prioritize being helpful.
**Brand Voice**: **Geeky Travel Pal**. Conversational, human, enthusiastic but controlled, and clear.
**Formality**: **Always use informal "tu"** for user address.
**Critical Exclusions**: 
- **NO "discount" or "guaranteed"**: Never use these terms. Use neutral phrasing like "price drops" or "deals".
- **NO superlatives**: Avoid "best", "cheapest ever", "greenest". Use subjective claims (e.g., "epic prices") only if non-comparative.
- **NO greenwashing**: Sustainability claims must be fact-based comparatives (e.g., "20% fewer emissions"). Never use "eco", "clean", or "sustainable" as absolute labels.
- **NO Title Case**: Use sentence case for all titles, UI elements, and CTAs.

---

## 1. Tone & Voice
### 1.1 Playful
- Write with a light, natural energy, as if talking to a friend. 
- **Use**: Travel hacks, clear explanations.
- **Optional (Sparingly)**: Rhymes, alliteration, metaphors (if natural).
- **Avoid**: "Fake-smart" corporate jargon, "LOL-funny" attempts, stiff/overly formal phrases.

### 1.2 Geeky
- Focus on data, patterns, and expertise.
- **Explain** complex info humanly.
- **Tone**: Enthusiastic but controlled (knowledgeable friend, not a lecturer).

### 1.3 Helpful
- **Prioritize the user**: Start with the benefit/instruction, then explain.
- **Be honest**: For bad news, drop playfulness, show empathy, be clear on next steps.
- **No cold apologies**: Avoid "We apologize for the inconvenience". Use sincere, solution-focused language.
- **Short & Simple**: Use short sentences. Prefer "pose question then offer solution".

---

## 2. Language-Specific Requirements (Romanian)
### 2.1 Grammar & Syntax
- **Possessives**: Reduce usage. They are often superfluous in Romanian (e.g., use "la următoarea călătorie" instead of "la următoarea ta călătorie").
- **Exclamation Marks**: **Use sparingly** (only for excitement or interjections). **NEVER use in instructions or direct user messages** (prevents "shouting" tone).
- **Commas**: Use only as required by grammar; do not add for pauses.
- **Capitalization**: 
  - **NO Title Case**. Use sentence case.
  - **No capital letter after colon**.
- **Quotes**: Use `« »` for inner quotes; use `""` for main quotes.
- **Ampersands**: Translate as `și` (unless part of a brand name).
- **Inclusive Language**: Avoid gendered assumptions. Use neutral phrasing (e.g., "Te-ai conectat" instead of "Ești conectat").

### 2.2 Formatting & Technical
- **Dates**: `DD.MM.YYYY`. Do not capitalize month names.
- **Time**: 24-hour format (e.g., `14:53`).
- **Numbers**: Use full stop `.` for thousands, comma `,` for decimals.
- **Currency**: `250 EUR`. Use non-breaking space. **Do not use symbols** (`€`) in text (graphic contexts only).
- **Hyphens**: No spaces before/after (e.g., `1-23 septembrie`).
- **Dashes**: Use for pauses (replace with comma/colon if possible). If a place name has a hyphen, use a dash preceded/followed by a hyphen (e.g., `București – Cluj-Napoca`).
- **Units**: Use metric system.

---

## 3. Technical & Placeholders
- **URL Requirements**: 
  - **All lowercase**.
  - **No special/extended characters** (replace with Latin base).
  - **Hyphens only** for word separation.
- **Placeholders/Variables**: **DO NOT translate, delete, or change**.
  - Keep exact order. Ensure sentence grammar works (use workarounds if needed).
  - *Example (Grammar workaround)*:
    - English: "Your trip to @@destination@@"
    - Romanian: "Călătoria ta cu destinația @@destination@@" (avoids preposition mismatch between cities/countries).

---

## 4. Brand & Content-Specific Rules
### 4.1 Feature Names & Brand
- **Skyscanner**: Never translate. Add modifiers like "platforma", "site-ul" if needed.
- **Feature Names**: Prioritize meaning over literal translation. Must be short, clear, and familiar to local users.
- **Best**: Use only when clear disclaimers are present (e.g., sorting results). Avoid otherwise.

### 4.2 UI & CTAs
- **Tone**: Short, clear, action-focused.
- **Structure**: One idea per string.
- **CTAs**: **Must use imperative form**.
- **Consistency**: Use identical wording for recurring elements (e.g., always "Price Alerts").

### 4.3 Marketing & SEO
- **Marketing**: Fluid, inspirational, natural. Avoid literal translations.
- **SEO**: Use natural search language. Avoid keyword stuffing.
- **Google Ads**: Very concise, clear benefits, direct CTA. No unprovable claims.

### 4.4 Sustainability
- **Strictly factual**: Use comparatives (e.g., "X is more sustainable than Y").
- **Avoid**: "Green", "natural", "clean", "eco", "guilt-free".
- **Data**: Only reference environmental performance when based on measurable data.

## 6. Examples (Correct vs. Incorrect)

**Correcting Tone & Style**
- **Do**: "În sfârșit ai un mod de a compara zborurile fără să-ți blochezi computerul."
- **Don't**: "Avertisment: instrumentul nostru... Ține-te bine și bucură-te de economii!" (Too excited/informal).

**Addressing Users**
- **Do**: "Vrei să afli în ce zi zborurile sunt cele mai ieftine? Te ajutăm noi."
- **Don't**: "Dacă încerci să îți dai seama... respiră și relaxează-te. De asta suntem aici." (Overly long/conversational).

**Capitalization & UI**
- **Do**: "Rezervă acum"
- **Don't**: "Rezervă Acum" (No title case).
- **Do**: "Afișează zboruri"
- **Don't**: "Aruncă o privire la câteva zboruri..." (Too wordy).

**Sustainability**
- **Do**: "Acest zbor are cu 20% mai puține emisii de CO2 decât un zbor obișnuit."
- **Don't**: "Acesta este un zbor sustenabil." (Vague/Greenwashing).

**Addressing "Best" / Superlatives**
- **Do**: "Compară oferte de zbor către Paris și găsește opțiunea potrivită pentru tine."
- **Don't**: "Găsește cele mai bune zboruri către Paris." (Avoid superlative "cele mai bune").
